# BudgetVault v3.3 — "Wedge + Foundation" Design Spec
**Date:** 2026-04-16
**Author:** Zach (with 18-agent audit synthesis)
**Status:** Draft for approval
**Audit source:** `docs/audit-2026-04-16/`

---

## 1. Vision

Ship two releases over 7–9 weeks that:
1. **v3.3.0 ("The Wedge")** — captures the 2026 privacy/AI-fatigue zeitgeist for maximum acquisition lift while fixing 5 production-critical bugs.
2. **v3.3.1 ("The Foundation")** — lands the iOS 18 migration, schema indexes, repository seam, design system, and CI/CD that v3.4 Households (CKShare + $24.99 tier) will require.

By the end of v3.3.1, BudgetVault should be:
- Cited by AI assistants for "best privacy-first iOS budget app"
- Shipping a viral Wrapped artifact that drives organic install
- Visibly using on-device Apple Intelligence (without sending data anywhere)
- Architecturally ready for partner sharing without a full rebuild
- Releasing on a one-command pipeline with privacy-safe crash telemetry
- Ranking on its own brand SERP (no more PWA squatter winning)

## 2. Non-Goals

These are **explicitly deferred** to v3.4 or later:

- CKShare partner sharing implementation (design only in v3.3.1, ship in v3.4)
- $24.99 Households tier (depends on CKShare)
- Split transactions (v3.4)
- Catch-up mode (v3.4)
- iPad UI (v3.4 — project.yml flag already trivial; UI adaptation is the work)
- Apple Watch app (v3.5+)
- Localization beyond DE metadata (v3.5+)
- Cash flow forecasting (v3.5+)
- Lifetime+ tier ($39.99) — wait for v3.4 data

## 3. Hard Constraints (non-negotiable)

- ❌ No subscription pricing — `$14.99 once` is the wedge
- ❌ No Plaid / bank sync — invalidates "Data Not Collected"
- ❌ No third-party analytics SDKs — same reason
- ❌ No cloud LLM APIs (OpenAI, Anthropic, Gemini) — same reason
- ❌ No Sentry / Crashlytics / Bugsnag — use MetricKit (Apple-first-party)
- ❌ No price reduction
- ❌ No mascot, no XP/levels, no streak-shame copy

## 4. Architecture Decisions

### 4.1 Two-Phase Release Strategy
**v3.3.0** ships in 4 weeks as the marketing-led release. **v3.3.1** ships 3–5 weeks later as the engineering-led release. This protects the marketing window (zeitgeist trends decay) while preventing the "ship architecture and features in the same release" failure mode.

### 4.2 BudgetVaultShared SPM Package
A new local Swift package `BudgetVaultShared/` containing types currently duplicated across app + widget + intents targets:
- `BudgetActivityAttributes`
- `WidgetData` (collapses `WidgetBudgetData` and `WidgetDataService.WidgetData`)
- `CurrencyFormatter`, `MoneyHelpers`
- `AppStorageKeys`
- `BudgetRepository` protocol (introduced in 4.3)
- `PremiumGate` enum (introduced in 4.4)

**Why now:** unblocks future Watch target (cannot import app target), prevents codec-drift bugs, lets Live Activity ship UI updates without rebuilding the main app.

### 4.3 BudgetRepository Protocol
A repository-pattern seam over SwiftData that:
- Exposes period-scoped reads (`transactions(in: DateInterval)`, `budget(for: Date)`)
- Memoizes `Category.spentCents(in:)` per render pass
- Lives in `BudgetVaultShared`
- Has a `LiveBudgetRepository` (SwiftData-backed) and `MockBudgetRepository` (test fixtures)

**Why now:** every audit (Code Review, Performance, Database, Architecture) flagged unbounded `@Query` as the #1 risk. Repository is the only seam that makes ViewModels unit-testable AND fixes the perf cliff AND enables iPad split-view (which renders heavy views simultaneously).

### 4.4 PremiumGate Enum
Centralizes 16 scattered `@AppStorage(isPremium)` reads with hardcoded magic numbers (`>= 6` categories, `>= 3` recurring, `> 4` import). API:
```swift
enum PremiumGate {
    static func canAddCategory(count: Int) -> Verdict
    static func canAddRecurring(count: Int) -> Verdict
    enum Verdict { case allowed, blocked(reason: String, paywallTrigger: PaywallTrigger) }
}
```

**Why now:** v3.4's $24.99 tier ladder needs to diff a 2-line enum, not 16 view files.

### 4.5 PaywallTrigger Enum
Replaces 8 separate `@State showPaywall = false` view-locals with one enum that drives contextual paywall headlines:
```swift
enum PaywallTrigger { case categoryLimit, recurringLimit, debtTracker, vaultIntelligence, wrapped, csvImport, rolloverRule, settings }
```
PaywallView's hero adapts: "Add unlimited envelopes — $14.99 once" instead of generic "Unlock the Full Vault."

**Why now:** UX audit ranked this as a top conversion lift; also gives team on-device-only signal for which trigger converts.

### 4.6 SchemaV2 — Additive Indexes Only
SchemaV2 ships in v3.3.1 with `#Index` annotations on the fields the existing `BudgetVaultSchemaV1.swift:4-39` checklist already enumerates:
- `Transaction(date)`, `Transaction(category, date)`, `Transaction(isIncome, date)`
- `Budget(year, month)`
- `RecurringExpense(isActive, nextDueDate)`

**Migration:** lightweight (no field changes, no data transforms). Adds the empty-V2 stage to `BudgetVaultMigrationPlan.stages` so V2→V3 (CKShare's sharing-root model) has a tested mechanism.

**Why now:** indexes were blocked on iOS 18; iOS 18 migration is in v3.3.1.

### 4.7 BRAND.md + BrandStrings.swift
A 1-page `docs/BRAND.md` codifies:
- Color tier rules (when to use neon vs. accent vs. semantic)
- Lock-only iconography rule
- Voice rules (calm, private, premium; no exclamation marks; vault verbs)
- One canonical privacy line

`BrandStrings.swift` registry centralizes user-facing copy currently scattered across NotificationService (7 strings), PaywallView (12 strings), SettingsView, etc.

### 4.8 Apple Intelligence Integration Strategy
- **NLEmbedding (iOS 17+)** for category suggestion — falls back to existing exact-match in `CategoryLearningService`. Ships in v3.3.1.
- **Foundation Models (iOS 18.1+)** for Wrapped narration — opt-in, gated by `SystemLanguageModel.availability`, falls back to existing static template copy. Ships in v3.3.1.
- All AI marketed as "patterns" not "AI" until Foundation Models ships.

### 4.9 CI/CD on Self-Hosted Mac (no third-party servers)
Fastlane + GitHub Actions runner on user's local Mac. Lanes:
- `test` — `xcodegen generate && xcodebuild test`
- `beta` — match → gym → pilot upload
- `release` — deliver metadata + screenshots

**Why self-hosted:** no source code or build artifacts leave the network. Privacy-brand-aligned. Free.

### 4.10 MetricKit — Privacy-Safe Crash Telemetry
`MXMetricManager` delivers crashes/hangs/disk/battery on-device only. Persist to `Documents/diagnostics/` (excluded from backup). Surface via existing `FeedbackService` so users can opt to attach when reporting bugs.

**Replaces:** Sentry/Crashlytics/Bugsnag. Apple-first-party = no privacy label impact.

## 8. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Phase 1 zeitgeist window decays before ship | Hard 4-week timebox; cut DE localization metadata first if slipping |
| iOS 18 migration breaks something subtle | Ship on a feature branch; full XCUITest run before merge; monitor MetricKit on TestFlight |
| Foundation Models latency too slow for Wrapped | Pre-generate all slides at sheet-open; static fallback always available |
| budgetvault.app squatter doesn't respond to UDRP | Ship `/vs/` pages and Wikidata entry anyway; wins on schema and recency signals |
| BudgetRepository refactor introduces regressions | Repository is parallel-introduced (views can use either); cut over view-by-view with full XCUITest each |
| Marketing-led release ships ahead of architecture release | Acceptable — that's the design intent. v3.3.0 must look indistinguishable from a major release to users; v3.3.1 is internal-quality release. |

## 9. Decision Log

- **Two-phase vs. monolithic v3.3:** chose two-phase to capture the zeitgeist window without forcing the schema migration into the same release.
- **NLEmbedding before Foundation Models:** ships sooner, lower latency, no opt-in burden — Foundation Models is the marketing moment.
- **CKShare design doc, not implementation:** the schema work needs to be right; rushing it to v3.3 risks a v3.3.2 emergency migration.
- **Self-hosted CI vs. GitHub-hosted:** self-hosted = no third-party servers = privacy-brand-aligned + free.
- **Static site rebuild vs. patch SPA:** static is the only way schema renders for AI crawlers; SPA patching is technical debt.
