### 8. REVIEW MANAGEMENT STRATEGY

BudgetVault already has a solid review prompt trigger in the spec (after under-budget month or 14+ day streak, suppressed 48h after paywall decline). Here is how to maximize the impact:

**Optimal Prompt Timing (in priority order):**

1. **After completing first under-budget month** -- This is peak satisfaction. The user has succeeded at the app's core promise. Conversion to 5-star review is highest here.

2. **At the 14-day streak milestone** -- User has formed a habit and is invested. Strong positive sentiment.

3. **After successfully importing data from CSV** -- User has committed to the app by migrating their data. High switching cost means high satisfaction signal.

4. **After viewing Monthly Summary with positive results** -- User just saw proof the app works for them.

**Timing rules already in spec (confirmed good):**
- Suppress for 48 hours after paywall decline (never ask upset users)
- Max 3 per year per Apple policy
- Never prompt on first launch or first week

**Additional recommendations:**

- **Never prompt after a transaction entry.** Users are in "get it done" mode. Interrupting the core flow creates negative sentiment.

- **Implement a "soft ask" before the system prompt.** Show a custom in-app dialog: "Are you enjoying BudgetVault?" with "Yes!" and "Not really" buttons. If "Yes" -- trigger SKStoreReviewController. If "Not really" -- show a feedback form (email/in-app) instead. This filters negative reviews into private feedback.

- **Respond to every App Store review** (especially negative ones) within 48 hours using App Store Connect's developer response feature. Template:
  - For bugs: "Thanks for the report. We've identified this issue and a fix is coming in the next update. Please reach out to support@budgetvault.app if you need immediate help."
  - For feature requests: "Great suggestion! We're tracking this for a future update. Thanks for helping shape BudgetVault."
  - For positive reviews: "Thank you! Reviews like yours help other privacy-conscious budgeters find us."

- **Track review-to-download correlation** in App Store Connect analytics. If average rating drops below 4.5, pause review prompts and investigate.

### 11. PRIVACY NUTRITION LABELS AS MARKETING ASSET

BudgetVault's minimal data collection is a MASSIVE competitive advantage in App Store presentation. Here is the analysis:

**BudgetVault's Privacy Label (projected):**

```
Data Not Collected
- BudgetVault does not collect any data.
```

Or at most (if using CloudKit):
```
Data Linked to You:
- None

Data Not Linked to You:
- Diagnostics (crash logs via Apple's standard framework)

Data Used to Track You:
- None
```

**Competitor Privacy Labels (for comparison):**

| App | Data Collected | Tracking |
|-----|---------------|----------|
| YNAB | Financial info, contact info, identifiers, usage data, diagnostics | Some tracking |
| Monarch Money | Financial info, contact info, location, identifiers, usage data | Tracking for analytics |
| Copilot | Financial info, identifiers, usage data, diagnostics | Some tracking |
| PocketGuard | Financial info, contact info, location, identifiers, usage data | Significant tracking |
| Goodbudget | Financial info, contact info, identifiers, usage data | Some tracking |
| EveryDollar | Financial info, contact info, identifiers, usage data | Some tracking |

**How to leverage this:**

1. **Screenshot callout**: Include a screenshot or visual element that shows the Privacy Nutrition Label comparison. A simple graphic: "We collect: Nothing. They collect: [long list]." (Do NOT name specific competitors -- show a generic "typical budget app" comparison.)

2. **Description mention**: Already included in the recommended description ("No third-party analytics or advertising SDKs").

3. **Promotional text rotation**: Use "Check our Privacy Nutrition Label -- we collect zero data" as a promotional text variant.

4. **App Preview Video**: Include a 2-second frame showing the privacy label as part of the trust-building sequence.

5. **Website/landing page**: Create a dedicated privacy comparison page that can be linked from the App Store description's support URL.

6. **Review responses**: When users praise privacy in reviews, respond highlighting the nutrition label: "We're proud of our 'Data Not Collected' privacy label -- it's a promise we take seriously."

**Critical implementation note**: To achieve the cleanest possible privacy label:
- Do NOT include any third-party analytics SDKs (no Firebase Analytics, no Mixpanel, no Amplitude)
- Do NOT include any crash reporting SDKs (use Apple's built-in crash reporting only)
- Do NOT include any advertising SDKs
- CloudKit usage with private database does NOT require declaring data collection (Apple considers it user-controlled)
- StoreKit 2 is Apple-native and does not add to privacy labels
- If you MUST have analytics, use Apple's App Analytics (built into App Store Connect) which requires zero in-app SDK
